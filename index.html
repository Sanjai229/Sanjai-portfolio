Speech Recognition and Gender Detection System

Problem & Motivation
Most speech-based applications demonstrate transcription using pre‑recorded datasets without addressing real-time microphone input or real‑world noise. I wanted to build an end-to-end system that captures live speech from a browser, transcribes it, and extracts speaker‑level insights such as gender under real usage conditions.

System Design
A browser audio recorder captures speech and sends it to a Flask backend, where the audio is standardised to WAV format. Whisper transcribes the speech, MFCC features are extracted, and a trained Random Forest model predicts speaker gender. Results are returned as structured JSON and rendered in UI.

Key Challenges
– Browser audio format inconsistency
– Noise & accent variation
– End-to-end debugging across frontend, backend, and models

Trade-offs & Learnings
I emphasised reliable data handling over optimising model accuracy. This helped me understand how real-world audio quality impacts inference behaviour more than algorithm choice.

Customer Churn Prediction Web Application

Problem & Motivation
Customer churn is a critical business concern, but many ML projects end at accuracy tables. I built a web churn predictor for real users, not just code notebooks.

System Design
A HTML form collects customer inputs and sends them to a FastAPI backend. The backend applies scaling and encoding, then generates churn probabilities with a trained Random Forest classifier shown via Jinja2 templates.

Key Challenges
– Feature name & ordering consistency
– Manual encoding/scaling in inference
– Deployment‑time parsing & dependency fixes

Trade-offs & Learnings
Explicit preprocessing and debugging pipelines reduced deployment issues, showing that most failures come from data mismatches, not the model itself.
