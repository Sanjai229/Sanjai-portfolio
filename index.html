Speech Recognition and Gender Detection System

Problem & Motivation
Most speech-based applications demonstrate transcription using pre-recorded datasets without addressing real-time microphone input or real-time insights. I wanted to build an end-to-end system that captures live speech from a browser, performs transcription, and generates useful information such as speaker gender under real-world audio constraints.

System Design
A browser-based recorder captures speech and sends it to a Flask backend. Audio is standardised into WAV format. Whisper is used for speech-to-text transcription, and MFCC features are extracted for speaker-level analysis. A trained Random Forest model predicts speaker gender. Results return as structured JSON and are rendered on the frontend.

Key Challenges
– Browser audio format inconsistency
– Background noise and accents
– End-to-end errors across frontend/backend/model integration

Trade-offs & Learnings
I prioritised robust data handling and system clarity over squeezing accuracy. This project improved my understanding of audio preprocessing, model integration, and real-world inference behaviour.

Customer Churn Prediction Web Application

Problem & Motivation
Customer churn is a critical business problem, but many machine learning projects stop at model accuracy and notebooks without demonstrating how predictions are consumed in real-world systems...

System Design
A HTML form collects customer inputs and sends them to a FastAPI backend. The backend scales and encodes features then uses a Random Forest classifier to generate churn probabilities rendered in Jinja2 templates.

Key Challenges
– Ensuring exact feature name & ordering consistency
– Handling encoding/scaling outside training pipeline
– Debugging deployment-time form parsing & dependencies

Trade-offs & Learnings
Explicit preprocessing improved robustness. Most ML deployment failures stem from data mismatches rather than model choice.
