Speech Recognition and Gender Detection System

Problem & Motivation
Most speech-based applications rely on offline datasets and avoid real-time audio capture challenges. This project focuses on building an end-to-end system that processes live browser-recorded audio while handling noise, format inconsistencies, and inference constraints.

System Design
Browser-recorded audio is sent to a Flask backend, standardised into WAV format, transcribed using Whisper, and processed to extract MFCC features. A trained Random Forest model predicts speaker gender, and results are returned as structured JSON and rendered dynamically on the frontend.

Result / Output
Real-time transcription with gender prediction from live microphone input.

Key Challenges
– Standardising browser-recorded audio for ML inference
– Maintaining MFCC feature consistency between training and inference
– Debugging failures across frontend, backend, and model layers

Trade-offs & Learnings
I prioritised robustness and system clarity over aggressive optimisation. The project highlighted how preprocessing and data handling dominate real-world ML performance more than model complexity.

Customer Churn Prediction Web Application

Problem & Motivation
Many churn prediction projects stop at notebooks and metrics. This project demonstrates how a trained ML model can be exposed through a production-style web interface for real-time business use.

System Design
A HTML form collects customer inputs and sends them to a FastAPI backend. Features are manually aligned using a pre-fitted StandardScaler and one-hot encoding to match the training schema. A Random Forest classifier returns churn predictions and probabilities rendered via Jinja2 templates.

Result / Output
Web-based churn prediction with probability output for business users.

Key Challenges
– Feature ordering and naming consistency during inference
– Manual preprocessing outside the training pipeline
– Deployment-time dependency and parsing issues

Trade-offs & Learnings
Explicit preprocessing improved debuggability and reliability. The project reinforced that deployment failures usually stem from data mismatches rather than model choice.
