Customer Churn Prediction Web Application

Problem & Motivation
Customer churn is a critical business problem, but many machine learning projects stop at model accuracy and notebooks without demonstrating how predictions are consumed in real-world systems. I wanted to build a production-style churn prediction application that not only trains a reliable ML model, but also exposes it through a public web interface where non-technical users can input customer details and receive interpretable churn predictions in real time.

System Design
The system follows a clear frontend–backend–model architecture. A HTML-based form collects customer attributes and sends them as a POST request to a FastAPI backend. The backend performs strict feature alignment, including numeric scaling using a pre-fitted StandardScaler and manual one-hot encoding to exactly match the model’s training schema. A trained Random Forest classifier is then used to generate churn predictions and probabilities. Results are rendered back to the frontend using Jinja2 templates and deployed as a public web service on Render.

Key Challenges
– Ensuring exact feature name and ordering consistency between training and inference
– Handling categorical encoding and numeric scaling outside the training pipeline
– Debugging deployment-time failures related to missing dependencies and form parsing
– Designing backend logic robust enough to handle partial or user-generated input

Trade-offs & Learnings
I prioritised correctness, debuggability, and production realism over rapid prototyping. This meant explicitly managing preprocessing steps instead of relying on implicit pipelines. The project deepened my understanding of how most ML deployment failures arise not from model choice, but from feature mismatches, data handling, and environment inconsistencies. It also strengthened my ability to debug full-stack ML systems across frontend, backend, and deployment layers.
